# Student Alert Automation Pipeline

[![Python 3.10+](https://img.shields.io/badge/python-3.10+-blue.svg)](https://www.python.org/downloads/)
[![Selenium](https://img.shields.io/badge/selenium-4.x-green.svg)](https://pypi.org/project/selenium/)
[![Ollama](https://img.shields.io/badge/ollama-local_AI-orange.svg)](https://ollama.com/)

An end-to-end, zero-touch Python automation script that seamlessly connects a Gmail inbox, a local AI model, and a remote web portal. 

It automatically downloads weekly class assignments directly from incoming emails, uses local pattern-matching and a local **Ollama AI (Gemma 3)** model to cleanly sort files into their respective class folders, compiles them into optimized zip archives, and finally utilizes a headless Chrome browser via **Selenium** to upload the final files to the *Student Alert* educational dashboard.

## ✨ Key Features
- **Hands-Free Email Parsing:** Plugs directly into a Gmail inbox using IMAP, actively searching for specific subjects (e.g., "Week 47").
- **Hybrid AI File Sorting:** Uses pre-defined Regex rules for immediate local file sorting. Any files with ambiguous or poorly formatted names are dynamically sent to a local Ollama AI model to be perfectly categorized—meaning 0% manual sorting.
- **Auto-Archiving:** Compiles sorted files into cleanly formatted `.zip` directories that align exactly with the remote web portal's expected structure.
- **Headless Web Automation:** Uses Selenium WebDriver to automatically log into the Student Alert portal, navigate the hidden dropdown DOM architecture, and securely upload the `.zip` files.
- **Bulletproof Reliability:** 
  - Complete with dynamic `WebDriverWait` checks (no hardcoded sleep racing).
  - Advanced standard Python `logging` directly to a terminal and `automation.log`.
  - Secure `.env` credential management.
  - Mathematically sound garbage collection using `tempfile` that guarantees no leftover junk files on your hard drive, even if the script crashes mid-upload.

## 🚀 Quick Start Guide

### 1. Prerequisites
You need the following installed on your system out of the box:
- **Python 3.10+**
- **Google Chrome** (The script uses standard Chrome for Selenium)
- **[Ollama](https://ollama.com/)** running locally. 
  - *Note: Ensure you pull the model first:* `ollama run gemma3:12b`

### 2. Installation
Clone the repository and install the required dependencies:

```bash
git clone https://github.com/yourusername/student-alert-automation.git
cd student-alert-automation

# Install the required Python packages
pip install selenium requests python-dotenv
```

### 3. Configuration
The script relies on a secure `.env` file to manage your Gmail App Passwords and web portal credentials. **Never commit this file to GitHub.** Create a file named `.env` in the root directory:

```env
# -> Gmail IMAP configuration (Use an App Password, not your standard password)
EMAIL_USER="your_gmail_address@gmail.com"
EMAIL_PASS="your_16_digit_app_password"

# -> Student Alert Portal credentials
SITE_USERNAME="your_portal_email@domain.com"
SITE_PASSWORD="your_portal_password"

# -> Toggle Browser Visibility
# True = Invisible background processing. False = Pop up Chrome so you can watch.
HEADLESS_MODE=False
```

### 4. Running the Automation
You can run the automation script at any time with a single command:
```bash
python autopilot.py
```

The script will:
1. Contact Gmail and download unread attachments matching the configured regular expressions.
2. Cross-reference naming conventions. If ambiguous (e.g., "Math assignment.pdf"), it passes a JSON structure to `http://localhost:11434` for Ollama to assign the correct Class folder.
3. Zip the sorted folders into a temporary workspace.
4. Launch Chrome (headlessly or visibly, depending on your `.env` config) and inject the zip files into the destination portal.
5. Save the processed Email IDs to `processed_log.txt` so it never duplicates work on a subsequent run.

## 🛠 Architecture Overview

### `autopilot.py`
The master script that handles the entire pipeline.
- `clean_filename()` / `classify_file_locally()`: Performs Phase-1 deterministic Regex sorting.
- `ask_ai_batch()`: The Phase-2 local webhook to Ollama. It enforces strict JSON layout returns for zero-hallucination processing.
- `process_email_and_upload()`: The core business logic tying IMAP, tempfile operations, zip archiving, and Selenium execution together.

### `.env`
The credentials store. Ensure this is continually added to your `.gitignore`.

### `automation.log`
Automatically generated by Python's `logging` module. Contains a permanent, timestamped history of every file processed, AI classification result, and any caught network timeouts (e.g., IMAP drops).

### `processed_log.txt`
A persistent local database containing unique Email IDs. Prevents the automation from downloading the exact same files twice if the script is run daily.
